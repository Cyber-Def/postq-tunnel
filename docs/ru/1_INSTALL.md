# Инструкция по запуску и настройке (PostQ-Tunnel)

## 0. Требования к инфраструктуре
- Установленный язык **Go 1.24** или выше (у вас установлен `Go 1.26`).
- **Публичный VPS-сервер** со свободным белым выделенным IP (например, базовая виртуалка на Ubuntu/Debian).
- **Доменное имя** (например, `yourdomain.com`), направленное A-записью на IP-адрес вашего VPS. 
  - *Важно:* Также потребуется настройка **Wildcard-записи** `*.yourdomain.com` (или CNAME), чтобы туннели могли динамически занимать любые поддомены (например, `api.yourdomain.com`).

## 1. Компиляция бинарников
Проект строится на парадигме *Zero-Dependency*, поэтому вам нужно всего лишь скомпилировать два легковесных бинарных файла. Внутри корневой папки проекта выполните:

```bash
# Сборка публичного Edge-Сервера (этот файл поедет на VPS)
/usr/local/go/bin/go build -o qtunnel ./cmd/server/main.go

# Сборка локального агента-клиента (этот файл остается на вашем компьютере/Mac)
/usr/local/go/bin/go build -o qtun ./cmd/qtun/main.go
```

## 2. Настройка VPS (публичный шлюз)
Перенесите скомпилированный файл `qtunnel` на ваш сервер. Этот сервер работает связующим прокси и автоматически выпускает валидные SSL-сертификаты.

На сервере задайте переменные и запустите процесс (рекомендуется создать systemd-unit или запустить в `screen`/`tmux`):
```bash
export QTUN_DOMAIN="tunnels.yourdomain.com"
export QTUN_EMAIL="admin@yourdomain.com"  # Нужен для регистрации сертификатов Let's Encrypt

# Внимание: для бинда публичных портов (80 и 443) обычно требуются права superuser (root).
sudo -E ./qtunnel
```
**Порты, которые ОБЯЗАТЕЛЬНО должны быть открыты в фаерволе (UFW/Iptables) на VPS:**
- `80` и `443` — Публичный HTTP/HTTPS (для ваших пользователей/браузера).
- `4443` — PQC TLS Транспорт (исключительно для внутренних защищенных соединений ваших qtun агентов).
- `9090` — (Опционально) сбор статистики Prometheus (`/metrics`).

## 3. Запуск туннеля локально (Ваш ПК/Mac)
На вашем ноутбуке (или системном блоке умного дома), где лежат ваши сервисы (БД, веб-сервер, Docker):

**A) Обычный мгновенный проброс порта (как ngrok)**
Хотите показать локальный React-сервер:
```bash
./qtun -server IP_ВАШЕГО_VPS:4443 -sub react -local localhost:3000
```
*(Результат: ваш проект доступен всем по SSL ссылке `https://react.tunnels.yourdomain.com`)*

**B) Проброс с защитой базовым паролем (Basic Auth)**
```bash
./qtun -server IP_ВАШЕГО_VPS:4443 -sub internal-api -local localhost:8080 -auth "admin:supersecret"
```
*(Только те, кто знает связку admin/supersecret, пройдут дальше Edge-сервера)*

**C) Проброс с фильтрацией по IP (White-List)**
Доступ только для устройств, подключенных к вашему корпоративному VPN:
```bash
./qtun -server IP_ВАШЕГО_VPS:4443 -sub admin-db -local localhost:5432 -allow-ip "192.168.1.0/24"
```
