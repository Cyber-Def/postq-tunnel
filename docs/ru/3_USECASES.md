# Реальные User Stories (Сценарии Использования)

*Как инженеры и ИИ-архитекторы решают ежедневные проблемы с помощью PostQ-Tunnel.*

## 🤖 Юзкейс 1: Защита локального AI Automation (n8n, Langflow)
**Контекст (Проблема):** 
Вы хостите сложную автоматизацию в [n8n](https://n8n.io/) или Langflow у себя на домашнем мощном Mac Studio. Вам критически необходимо принимать внешние вебхуки (например, уведомления об оплатах из Stripe или сообщения из Telegram Bot API).
Но если вы выставите порт n8n в публичный интернет через обычный ngrok, хакеры найдут адрес сканером и смогут войти в вашу веб-админку, получив возможность выполнять произвольный код (RCE) на вашем маке.
**Решение:**
Мы защищаем туннель с помощью встроенного IP-вайтлиста и пароля.
```bash
# Даем доступ к UI только локальным IP или офисному WireGuard, и накидываем Auth:
./qtun -server yourvps.com:4443 -sub n8n -local localhost:5678 -allow-ip "198.51.100.0/24" -auth "admin:superHardPass"
```
**Итог:** Админ-панель спрятана за стеной Edge-сервера. Если Телеграму надо слать вебхуки без пароля, вы в будущем легко сможете добавить маршрутизацию `/webhook` мимо BasicAuth, оставив `/` запароленным. Но ваша система уже изолирована Zero-Trust подходом.

## 🧠 Юзкейс 2: Безопасный доступ к персональным LLM (Open WebUI / OpenClaw)
**Контекст (Проблема):**
У вас на Mac Studio крутится приватный ChatGPT-аналог (например, Llama 3) через Open WebUI на порту 3000. Это жрет GPU. Вы едете в такси и хотите воспользоваться своей нейросетью с iPhone, но не хотите, чтобы вашу ссылку нашли боты и майнили на вашей видеокарте.
**Решение:**
Поднимаем запароленный туннель на свой мобильный телефон.
```bash
./qtun -sub mygpt -local localhost:3000 -auth "aimaster:secureai"
```
**Итог:** Заходя в браузер Safari на телефоне (`https://mygpt.tunnels...`), сервер просит ввести логин и пароль. Трафик зашифрован устойчивой квантовой криптографией (`ML-KEM`). Провайдеры видят только белый шум.

## 🛠️ Юзкейс 3: "Wormhole" во внутреннюю базу данных 
**Контекст (Проблема):**
Вы разработчик в банке или аутсорс-компании. Тестовая (staging) база данных PostgreSQL не имеет выхода в интернет (находится за строгим корпоративным NAT). Подрядчику срочно нужно подключиться к БД со своего ноутбука из дома.
**Решение:**
Из офисной сети (которая имеет выход в наружный интернет, но запрещает входящие соединения) вы запускаете qtun к вашему настроенному VPS:
```bash
./qtun -sub testdb -local localhost:5432 -auth "devops:token"
```
**Итог:** Внешний DevOps подключает свой клиент (DataGrip/DBeaver) к `https://testdb.tunnels...` (или по сырому TCP). Внутренняя сеть не скомпрометирована открытием портов.

## 📱 Юзкейс 4: Быстрое Кросс-Тестирование Дизайна (Cross-Device Preview)
**Контекст (Проблема):**
Вы верстаете фронтенд нового продукта (React/Vite). Он крутится на `localhost:5173`. Вам нужно посмотреть, как анимации выглядят на старом iPad и на Android руководителя, который сидит в другом офисе. 
**Решение:**
Одной командой открываем проект:
```bash
./qtun -sub preview -local localhost:5173
```
Как только коллеги протестировали UI с телефона через мобильный 4G, вы нажимаете `Ctrl+C` в терминале мака.
**Итог:** Туннель мгновенно умирает. Graceful Shutdown на публичном сервере стирает маршрут за миллисекунды. Секретный проект больше недоступен. Никаких Deploy-ов в Vercel / Netlify. Без мусора.
