# 真实用户故事（使用场景）

## 🤖 场景 1: 保护本地企业级自动化工作流 (n8n, Langflow)
**痛点所在:** 你在本地部署了极耗性能的 n8n，为处理来自 Stripe / Telegram 的自动化回调，须开放公网访问接口。将整个面板暴露在公网极易遭受各类自动化恶意程序的扫描攻击。
**解决方案:**
```bash
./qtun -sub n8n -local localhost:5678 -allow-ip "198.51.100.0/24" -auth "admin:superHardPass"
```
**结果:** 一键构建零信任网关，无缝接听内部回传。

## 🧠 场景 2: 旅途中手机安全访问私有大语言模型 (Ollama / Open WebUI)
**痛点所在:** 你的 Llama 3 服务器跑在家庭百兆专线上。你想在手机端进行交互，但不希望外人偷跑你的 GPU 显卡。
**解决方案:** 为个人手机挂载密码加密专属网络。
```bash
./qtun -sub mygpt -local localhost:3000 -auth "aimaster:secureai"
```

## 🛠️ 场景 3: 内部网络数据库的 "透明虫洞"
轻松绕过企业内部严格的 NAT 环境，为外部聘请的临时协作工程师提供一条访问 `5432` PostgreSQL 测试库的安全流隧道。彻底杜绝内网直连所带来的合规性安全隐患。

## 📱 场景 4: 快速的前端跨设备动效预览
开发者可以通过移动数据网络 (4G/5G) 用移动设备展示 `localhost:5173` React/Vite 界面给异地长官。预览结束只需敲击 `Ctrl+C` 断开本地服务，云端隧道即时摧毁，没有繁琐的线上服务善后操作。
