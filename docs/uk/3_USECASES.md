# Реальні User Stories (Сценарії використання)

*Як саме інженери та AI-архітектори вирішують завдання щодня.*

## 🤖 Юзкейс 1: Захист локального AI Automation (n8n, Langflow)
**Проблема:** Ви налаштували автоматизацію в n8n локально. Вам потрібно приймати сторонні вебхуки з Stripe чи Telegram. Якщо просто відкрити порт, є ризик взлому системи та виконання коду (RCE) на вашому Mac.
**Рішення:**
Захист за допомогою IP-вайтлістів та паролів.
```bash
./qtun -server yourvps.com:4443 -sub n8n -local localhost:5678 -allow-ip "198.51.100.0/24" -auth "admin:superHardPass"
```
**Підсумок:** Веб-адмінка надійно захищена шлюзом. Хакери відсікаються ще на етапі сервера.

## 🧠 Юзкейс 2: Безпечний доступ до приватних LLM (Open WebUI)
**Проблема:** Ви хостите велику LLM через Ollama + Open WebUI. Хочете потестувати з iPhone в метро, але не бажаєте, щоб чужі користувачі використовували вашу відеокарту.
**Рішення:**
Підіймаємо тунель із вбудованою Basic Auth:
```bash
./qtun -sub mygpt -local localhost:3000 -auth "aimaster:secureai"
```
**Підсумок:** Доступ працює через 4G. Браузер на телефоні попросить пароль. Безпека рівня Enterprise.

## 🛠️ Юзкейс 3: "Wormhole" до внутрішньої бази даних
**Проблема:** Staging БД (PostgreSQL) не має прямого виходу в інтернет. Запрошеному девопсу потрібен тимчасовий доступ з дому.
**Рішення:**
З комп'ютера у внутрішній мережі запускається qtun:
```bash
./qtun -sub testdb -local localhost:5432 -auth "devops:token"
```
**Підсумок:** Девопс підключається через свій DataGrip. Внутрішня мережа залишилася закритою для іншого світу.

## 📱 Юзкейс 4: Швидкий Cross-Device Frontend Preview
**Проблема:** Зробили класний дизайн на React (`localhost:5173`). Треба на 3 хвилини показати замовнику анімацію на iPhone по 4G.
**Рішення:**
```bash
./qtun -sub preview -local localhost:5173
```
**Підсумок:** Після тесту натискаєте `Ctrl+C`. Тунель вмирає. Жодного сліду на жодних Vercel серверах не залишається.
