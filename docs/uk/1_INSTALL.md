# Інструкція із запуску та налаштування (PostQ-Tunnel)

## 0. Вимоги до інфраструктури
- Встановлено мову **Go 1.24** або вище (у вас встановлено `Go 1.26`).
- **Публічний VPS-сервер** з виділеною білою IP-адресою (наприклад, базова віртуальна машина на Ubuntu/Debian).
- **Доменне ім'я** (наприклад, `yourdomain.com`), спрямоване A-записом на IP-адресу вашого VPS. 
  - *Важливо:* Також знадобиться налаштування **Wildcard-запису** `*.yourdomain.com` (або CNAME), щоб тунелі могли динамічно отримувати будь-які піддомени.

## 1. Компіляція бінарників
Проєкт базується на парадигмі *Zero-Dependency*, тому вам потрібно лише скомпілювати два легких файли. У кореневій папці проєкту виконайте:

```bash
# Збірка публічного Edge-сервера (цей файл піде на VPS)
/usr/local/go/bin/go build -o qtunnel ./cmd/server/main.go

# Збірка локального агента-клієнта (цей файл залишається на вашому Mac/ПК)
/usr/local/go/bin/go build -o qtun ./cmd/qtun/main.go
```

## 2. Налаштування VPS (публічний шлюз)
Перенесіть скомпільований файл `qtunnel` на ваш сервер. Цей сервер працює як злучний проксі та автоматично видає валідні SSL-сертифікати.

На сервері задайте змінні та запустіть процес (рекомендується `systemd` або `tmux`):
```bash
export QTUN_DOMAIN="tunnels.yourdomain.com"
export QTUN_EMAIL="admin@yourdomain.com"  # Потрібен для Let's Encrypt

# Увага: для прив'язки до портів (80 і 443) потрібні права root.
sudo -E ./qtunnel
```
**Порти, що ОБОВ'ЯЗКОВО повинні бути відкриті у фаєрволі на VPS:**
- `80` і `443` — Публічний HTTP/HTTPS (для користувачів/браузера).
- `4443` — Транспорт PQC TLS (виключно для безпечних з'єднань ваших qtun агентів).
- `9090` — (Опціонально) збір статистики Prometheus (`/metrics`).

## 3. Запуск тунелю локально (Ваш ПК/Mac)
На вашому комп'ютері:

**A) Звичайний швидкий прокид порту (як ngrok)**
```bash
./qtun -server IP_ВАШОГО_VPS:4443 -sub react -local localhost:3000
```
*(Ваш проєкт доступний за: `https://react.tunnels.yourdomain.com`)*

**B) Прокид із базовим паролем (Basic Auth)**
```bash
./qtun -server IP_ВАШОГО_VPS:4443 -sub internal-api -local localhost:8080 -auth "admin:supersecret"
```

**C) Прокид з фільтрацією по IP (White-List)**
Доступ тільки для вашого корпоративного VPN:
```bash
./qtun -server IP_ВАШОГО_VPS:4443 -sub admin-db -local localhost:5432 -allow-ip "192.168.1.0/24"
```
